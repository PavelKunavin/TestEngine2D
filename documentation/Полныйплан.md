# TestEngine2D — план разработки

## 📌 Мета

- **Проект**: TestEngine2D — 2D-надстройка над TestEngine
- **Игра**: TestGame (киберпанк action-RPG)
- **Стиль**: Kohi-like, модули в `engine2d/`, префикс `te2d_`
- **Графика**: 2D-спрайтовая с элементами передовой графики (шейдеры, освещение, пост-эффекты)
- **Сборка**: отдельный `Makefile.2d`, изолирован от 3D-сборки TestEngine
- **Рендер**: переиспользует OpenGL-бэкенд из `engine/render/`

## 🔗 Зависимости от TestEngine (engine/)

| Модуль              | Использование в TestEngine2D                |
|---------------------|---------------------------------------------|
| `engine/platform/`  | окно GLFW, ввод, главный цикл, время        |
| `engine/math/`      | `te_vec3`, `te_mat4` (матрицы проекции)     |
| `engine/render/`    | OpenGL-бэкенд для 2D-рендера                |
| `vendor/glad`       | OpenGL loader                               |
| `vendor/glfw`       | оконная система                             |
| `vendor/imgui`      | отладочный UI                               |

**Не используются**: `engine/physics/`, `engine/animation/` (3D-специфика).

## 🚀 Этапы разработки

### Этап 2D.0 — Фундамент и первый спрайт на экране

**Цель**: инициализация 2D-рендера, загрузка текстуры, отрисовка спрайта, камера.

**Задачи**:
1. Создать `te2d_types.hpp` — `te2d_vec2`, `te2d_rect`, `te2d_color`, `te2d_layer`.
2. Создать `te2d_texture` — загрузка PNG через stb_image, загрузка в OpenGL (GL_TEXTURE_2D).
3. Создать `te2d_sprite` — привязка текстуры + uv-регион.
4. Создать `te2d_sprite_batch` — dynamic VBO, батчинг квадов, сортировка по слою/текстуре.
5. Создать `te2d_camera` — ортографическая проекция, матрица view, плавное следование.
6. Создать `te2d_renderer` — init/shutdown/begin/end/flush, интеграция с `te_render_*`.
7. Шейдеры `sprite.vert` / `sprite.frag` в `assets2d/shaders/`.
8. `app2d/main.cpp` — инициализация, загрузка тестового спрайта, цикл с отрисовкой.

**Результат**: окно 1280×720, один спрайт на экране, камера двигается стрелками, ImGui-панель с FPS.

---

### Этап 2D.1 — 2D-физика

**Цель**: гравитация, движение, коллизии AABB.

**Задачи**:
1. `te2d_rigidbody` — позиция, velocity, acceleration, масса, drag, флаги (static/dynamic).
2. `te2d_collider` — AABB, Circle. Функции получения мирового AABB.
3. `te2d_collision` — AABB vs AABB, AABB vs Circle, разрешение (выталкивание).
4. `te2d_physics_world` — гравитация, список тел, шаг с фиксированным dt, интеграция.
5. Тест: спрайт падает под гравитацией, стоит на платформе (static collider).

**Результат**: персонаж (квадрат) падает, приземляется на платформу, двигается влево-вправо.

---

### Этап 2D.2 — Сцена и игровые объекты

**Цель**: система сущностей, объединяющая спрайт, физику и transform.

**Задачи**:
1. `te2d_transform` — позиция, поворот, масштаб, матрица модели (model matrix).
2. `te2d_entity` — контейнер компонентов: transform*, sprite*, rigidbody*, collider*.
3. `te2d_scene` — список сущностей, update(dt), draw(), добавление/удаление.
4. Интеграция с физикой: автоматическая синхронизация transform ↔ rigidbody.
5. Тест: персонаж из двух спрайтов (тело + голова), двигается и прыгает.

**Результат**: игровой объект с физикой и визуалом, управляемый с клавиатуры.

---

### Этап 2D.3 — 2D-анимация

**Цель**: покадровая анимация из спайсшита.

**Задачи**:
1. `te2d_animation` — массив кадров (uv-регионы + длительность), загрузка из JSON.
2. `te2d_animator` — текущая анимация, смена состояний (idle → run → jump), ручной переход.
3. Интеграция с `te2d_sprite` — uv подменяется каждый кадр.
4. Тест: персонаж анимируется в idle и при ходьбе.

**Результат**: персонаж с 4–6 кадрами анимации в разных состояниях.

---

### Этап 2D.4 — Визуальные эффекты

**Цель**: частицы и задел на пост-эффекты.

**Задачи**:
1. `te2d_particles` — эмиттер, атлас частиц, параметры (lifetime, velocity, color over time).
2. Интеграция в `te2d_sprite_batch` — частицы как отдельный слой.
3. `te2d_postprocess` — рендер в FBO, bloom (размытие + смешивание).
4. Шейдеры для bloom.
5. Тест: искры при ударе, неоновое свечение.

**Результат**: эффекты частиц и bloom, характерные для киберпанк-стиля.

---

### Этап 2D.5 — Тайловая карта и уровни

**Цель**: загрузка уровней из Tiled (TMX), рендер тайлов.

**Задачи**:
1. `te2d_tileset` — текстура + сетка тайлов, uv-регионы для каждого тайла.
2. `te2d_tilemap` — сетка, загрузка TMX (парсинг XML/JSON), отрисовка слоёв.
3. Коллизии с тайлами (платформы, стены).
4. Тест: уровень из Tiled с платформами, персонаж бегает и прыгает.

**Результат**: играбельный прототип уровня.

## 📝 Примечания

- **Префикс `te2d_`** для всех типов и функций 2D-надстройки.
- **Сборка**: `Makefile.2d` → `build\2DGame.exe`. Переключение — переименованием Makefile-ов.
- **Точка входа**: `app2d/main.cpp` (явно указан в Makefile.2d, без wildcard).
- **Ресурсы**: `assets2d/` отделены от `assets/`.
- **Рендер**: использует существующий `engine/render/` как основу.
- **Стиль**: Kohi-like — POD-типы, свободные функции, минимум зависимостей, ручное управление.
- **В этом диалоге**: только планирование. Код — в отдельном диалоге.
